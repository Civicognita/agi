# Configuration Reference

Aionima is configured via two files: `aionima.json` (structure and behavior) and `.env` (secrets). This document covers every available configuration field, how `.env` integration works, and how hot-reload operates.

---

## File Locations

| File | Location | Purpose |
|------|---------|---------|
| `aionima.json` | `~/.agi/aionima.json` | All configuration |
| `.env` | `~/.agi/.env` | Secrets (tokens, API keys) |

The gateway looks for `aionima.json` in the current working directory by default. Override with `--config <path>` on the CLI.

---

## Secrets Interpolation

Any string value in `aionima.json` can reference an environment variable using `$ENV{VARIABLE_NAME}` syntax:

```json
{
  "auth": {
    "tokens": ["$ENV{AUTH_TOKEN}"]
  }
}
```

At load time, `$ENV{AUTH_TOKEN}` is replaced with the value of the `AUTH_TOKEN` environment variable. Variables are read from the process environment, which includes values loaded from `.env` at startup.

Never put raw API keys or tokens directly in `aionima.json`. The `aionima doctor` command checks for common secret patterns and warns if it finds them.

---

## .env File

The `.env` file is loaded by the gateway at startup using a simple `KEY=VALUE` parser. It must have `0600` permissions (readable only by the owner).

```bash
# Aionima environment — secrets (mode 0600)
# Generated by `aionima setup`

AUTH_TOKEN=64-char-random-hex-string
ANTHROPIC_API_KEY=sk-ant-api03-...
TELEGRAM_BOT_TOKEN=123456789:AAF...
DISCORD_BOT_TOKEN=MTIzNDU2...
GMAIL_CLIENT_ID=123456789-xxxx.apps.googleusercontent.com
GMAIL_CLIENT_SECRET=GOCSPX-...
GMAIL_REFRESH_TOKEN=1//...
SIGNAL_API_URL=http://localhost:8080
WHATSAPP_ACCESS_TOKEN=EAA...
WHATSAPP_PHONE_NUMBER_ID=123456789012345
JWT_SECRET=64-char-random-hex-string
```

Set permissions:

```bash
chmod 0600 /opt/aionima/.env
```

## Full Configuration Schema

### gateway

Controls the HTTP server binding and gateway operational state.

```json
{
  "gateway": {
    "host": "127.0.0.1",
    "port": 3100,
    "state": "ONLINE"
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `host` | string | `"127.0.0.1"` | Listen address. Use `"0.0.0.0"` to expose on all interfaces. |
| `port` | number | `3100` | HTTP port |
| `state` | enum | `"OFFLINE"` | Starting state: `ONLINE`, `LIMBO`, `OFFLINE`, or `UNKNOWN` |

### auth

Authentication and rate limiting for the gateway API.

```json
{
  "auth": {
    "tokens": ["$ENV{AUTH_TOKEN}"],
    "maxAttemptsPerWindow": 10,
    "rateLimitWindowMs": 60000,
    "lockoutDurationMs": 300000,
    "maxBodyBytes": 2097152
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `tokens` | string[] | `[]` | Bearer tokens that grant access |
| `password` | string | — | Optional password-based auth |
| `maxAttemptsPerWindow` | number | `10` | Max failed auth attempts per IP before lockout |
| `rateLimitWindowMs` | number | `60000` | Auth rate limit window (ms) |
| `lockoutDurationMs` | number | `300000` | Lockout duration after too many failures (ms) |
| `maxBodyBytes` | number | `2097152` | Max request body size (2 MB) |

Loopback addresses (`127.0.0.1`, `::1`) bypass auth checks when no token is configured.

### agent

LLM provider configuration.

```json
{
  "agent": {
    "provider": "anthropic",
    "model": "claude-sonnet-4-6",
    "maxTokens": 8192,
    "maxRetries": 3,
    "replyMode": "autonomous",
    "devMode": false
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `provider` | enum | `"anthropic"` | LLM provider: `anthropic`, `openai`, `ollama` |
| `model` | string | `"claude-sonnet-4-6"` | Model identifier |
| `maxTokens` | number | `8192` | Max response tokens |
| `maxRetries` | number | `3` | Retry attempts on transient errors |
| `baseUrl` | string | — | Base URL for Ollama or proxy |
| `replyMode` | enum | `"autonomous"` | `autonomous` or `human-in-loop` |
| `devMode` | boolean | `false` | Enable developer identity and workspace context |
| `providers` | array | — | Failover provider list |
| `resourceId` | string | `"$A0"` | COA resource identifier |
| `nodeId` | string | `"@A0"` | COA node identifier |

### channels

Array of channel configurations.

```json
{
  "channels": [
    {
      "id": "telegram",
      "enabled": true,
      "config": {
        "botToken": "$ENV{TELEGRAM_BOT_TOKEN}"
      }
    }
  ]
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `id` | string | required | Channel plugin ID |
| `enabled` | boolean | `true` | Whether this channel is active |
| `config` | object | — | Channel-specific configuration (varies by channel) |

### owner

Owner identity and access configuration.

```json
{
  "owner": {
    "displayName": "Owner",
    "dmPolicy": "pairing",
    "channels": {
      "telegram": "123456789",
      "discord": "987654321098765432",
      "gmail": "you@example.com",
      "signal": "+15555550100",
      "whatsapp": "+15555550100"
    }
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `displayName` | string | `"Owner"` | Display name injected into the system prompt |
| `dmPolicy` | enum | `"pairing"` | `pairing` (code required) or `open` (allow all) |
| `channels` | object | `{}` | Per-channel owner user IDs |

### entities

SQLite entity store configuration.

```json
{
  "entities": {
    "path": "./data/entities.db"
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `path` | string | `"./data/entities.db"` | Path to the SQLite database file |

> **Note:** The schema default shown above is `./data/entities.db`. In production, the runtime config places the database at `~/.agi/entities.db`. The `aionima setup` wizard writes the production path automatically.

### queue

Message queue processing.

```json
{
  "queue": {
    "pollIntervalMs": 100,
    "concurrency": 10,
    "drainTimeoutMs": 5000
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `pollIntervalMs` | number | `100` | Queue poll interval |
| `concurrency` | number | `10` | Max concurrent message processing |
| `drainTimeoutMs` | number | `5000` | Shutdown drain timeout |

### sessions

Agent session management.

```json
{
  "sessions": {
    "contextWindowTokens": 200000,
    "idleTimeoutMs": 86400000,
    "maxSessions": 5000
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `contextWindowTokens` | number | `200000` | Token budget per session |
| `idleTimeoutMs` | number | `86400000` | Session idle timeout (24 hours) |
| `maxSessions` | number | `5000` | Max concurrent sessions |

### skills

Skills system configuration.

```json
{
  "skills": {
    "directory": "./skills",
    "watchForChanges": false
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `directory` | string | `"./skills"` | Directory containing `.skill.md` files |
| `watchForChanges` | boolean | `false` | Hot-reload skills on file changes |

### memory

Composite memory adapter configuration.

```json
{
  "memory": {
    "directory": "./data/memory"
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `directory` | string | `"./data/memory"` | Directory for file-based memory storage |

### workspace

Developer workspace configuration.

```json
{
  "workspace": {
    "root": ".",
    "projects": ["/home/user/projects"],
    "selfRepo": "/home/user/projects/aionima",
    "webhookSecret": "$ENV{WEBHOOK_SECRET}"
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `root` | string | `"."` | Workspace root for file tools |
| `projects` | string[] | `[]` | Directories where projects are hosted |
| `selfRepo` | string | — | Path to this repo (enables dashboard update detection) |
| `webhookSecret` | string | — | GitHub webhook HMAC secret |

### voice

Voice pipeline configuration.

```json
{
  "voice": {
    "enabled": false,
    "sttProvider": "whisper",
    "ttsProvider": "edge",
    "whisperModel": "whisper-1"
  }
}
```

See the [Voice Pipeline](./voice.md) document for full field details.

### hosting

Project hosting configuration.

```json
{
  "hosting": {
    "enabled": false,
    "lanIp": "192.168.0.144",
    "baseDomain": "ai.on",
    "portRangeStart": 4000,
    "containerRuntime": "podman"
  }
}
```

See the [Project Hosting](./project-hosting.md) document for full field details.

### logging

Log output configuration.

```json
{
  "logging": {
    "level": "info",
    "logDir": "./logs",
    "maxFileSize": 10485760,
    "maxFiles": 5,
    "stdout": true
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `level` | enum | `"info"` | Log level: `debug`, `info`, `warn`, `error` |
| `logDir` | string | `"./logs"` | Log file directory |
| `maxFileSize` | number | `10485760` | Max log file size (10 MB) before rotation |
| `maxFiles` | number | `5` | Max rotated log files to keep |
| `stdout` | boolean | `true` | Also write logs to stdout |

### heartbeat

Autonomous heartbeat configuration.

```json
{
  "heartbeat": {
    "enabled": false,
    "intervalMs": 3600000,
    "promptPath": "./data/persona/HEARTBEAT.md"
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | boolean | `false` | Enable autonomous heartbeat |
| `intervalMs` | number | `3600000` | Interval between heartbeats (1 hour) |
| `promptPath` | string | `"./data/persona/HEARTBEAT.md"` | Path to heartbeat prompt file |

### dashboardAuth

Multi-user dashboard authentication.

```json
{
  "dashboardAuth": {
    "enabled": false,
    "jwtSecret": "$ENV{JWT_SECRET}",
    "sessionTtlMs": 86400000
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | boolean | `false` | Enable multi-user auth |
| `jwtSecret` | string | — | Secret for signing session tokens |
| `sessionTtlMs` | number | `86400000` | Session TTL (24 hours) |

### prime

PRIME knowledge corpus location. Defaults to the system-level repo at `/opt/aionima-prime`.

```json
{
  "prime": {
    "dir": "/opt/aionima-prime",
    "source": "git@github.com:Civicognita/aionima.git",
    "branch": "main"
  }
}
```

### workers

Taskmaster engine settings. Workers are registered by plugins and discovered at boot.

```json
{
  "workers": {
    "autoApprove": false,
    "maxConcurrentJobs": 3,
    "workerTimeoutMs": 300000,
    "modelOverrides": {
      "code.hacker": {
        "provider": "anthropic",
        "model": "claude-opus-4-6"
      }
    }
  }
}
```

## Hot-Reload

The gateway watches `aionima.json` for changes using a file watcher. When the file changes:

1. The new config is loaded and validated against the Zod schema.
2. If valid, the `config:changed` hook is fired for each changed key.
3. Plugins and internal components that registered `config:changed` handlers update their state.
4. Channel adapters that support hot-reload pick up new settings without restarting.

Not all config changes hot-reload cleanly. Changes to `gateway.host`, `gateway.port`, and `entities.path` require a full gateway restart.

Changes to channel credentials (bot tokens, API keys) require the affected channel to be restarted. Use the dashboard channel controls to stop and start the channel after updating credentials.

## Example: Minimal Config

```json
{
  "gateway": {
    "host": "127.0.0.1",
    "port": 3100,
    "state": "ONLINE"
  },
  "auth": {
    "tokens": ["$ENV{AUTH_TOKEN}"]
  },
  "agent": {
    "provider": "anthropic",
    "model": "claude-sonnet-4-6",
    "replyMode": "autonomous"
  },
  "owner": {
    "displayName": "Owner",
    "dmPolicy": "pairing",
    "channels": {}
  },
  "channels": [],
  "entities": {
    "path": "./data/entities.db"
  },
  "workspace": {
    "root": "."
  },
  "logging": {
    "level": "info"
  }
}
```

This is the content of `aionima.example.json` — the starting point for a new installation. Run `aionima setup` to generate a populated version with production paths (e.g., `~/.agi/entities.db` instead of `./data/entities.db`).
